연속 성공일·완료율 그래프가 "내 멘탈은 내려갔지만 루틴은 올랐다"를 숫자로 증명.

나만의 규칙

"뉴스·거래량 폭발 시 24시간 대기" 같은 물타기 금지 규칙을 저장, Pro는 무제한/복원.

원페이지 배치 가이드

  • 헤더 우측 CTA 2버튼(Primary/Outline) 스티키 유지.
  • 히어로: 메시지+서브+앱 목업, CTA 반복 배치.
  • 기능 3~4카드 → 신뢰(데이터 출처/로컬 저장) → 구독 플랜(Monthly/Yearly).
  • FAQ/문의 버튼을 푸터 바로 위에 배치해 유입 손실 최소화.

톤 & UI

  • 딥 네이비/차콜 배경 + 라임 포인트, CTA는 그라데이션.
  • 1200px 컨테이너, 섹션 패딩 100px, 12그리드 4-4-4 카드.
  • 모바일: CTA 상단 고정, 카드 1열 스택, 여백 넉넉히.
  • 숫자/지표는 모노스페이스, 아이콘은 라인 스타일로 대비.

About Serious Lab

데이터와 모델로 생활을 덜 구리게 만드는 랩 팀

우리는 "원래 이렇게 쓰는 거예요"를 듣는 순간, 어디서 시간과 에너지가 새는지 계측부터 합니다. 불편함을 구경만 하느니, 로그를 남기고 수치를 뽑는 편이 재미있으니까요.

LLM이 모든 걸 해결해 준다는 말은 믿지 않습니다. 모델은 싸고 빠른 인턴일 뿐이고, 진짜 일은 데이터 정리와 피드백 루프에 있습니다.

Serious Lab은 생활의 자잘한 버그를 계량하고, LLM/AI를 붙여보고, 개선률을 공유합니다. 목표는 '덜 구린 생활', 태도는 시니컬, 결과는 실용적입니다.

Quantified Life
시간·돈·멘탈이 어디서 새는지 숫자로 본 뒤에야 고칩니다.
LLM Ops, Not Magic
프롬프트 장인보다 데이터 정합, 실패 로그, 재현성을 챙깁니다.
Human-in-the-loop
자동화는 좋아하지만, 마지막 품질 잣대는 사람입니다. 이상하면 즉시 끊습니다.

Philosophy

Measure, Mock, Iterate

문제를 감으로만 보지 않습니다. 계측 → 가설 → 모델 → 실험 → 회복/개선률 확인의 루프를 반복합니다. 모델을 믿기보단 의심하고, 숫자 앞에서는 집착합니다.

01

계측부터

현재 얼마나 안 좋은지 모르면, 좋아졌는지도 모릅니다. 로그와 베이스라인을 먼저 만듭니다.

02

모델은 인턴

LLM은 초안 작성, 분류, 요약에 좋은 값싼 인턴입니다. 결정권은 주지 않습니다.

03

사람이 브레이크

Human-in-the-loop는 의무입니다. 모델이 엉뚱한 소리하면 즉시 끊고, 학습 데이터에 다시 반영합니다.

04

숫자로 선택

기분 좋은 경험보다 개선폭·회복률·재현성을 보고 선택합니다. 안 움직이는 아이디어는 버립니다.

Lab Notes

데이터로 생활을 만지작거리다 생긴 로그

결과물 자랑보다 데이터에 부딪히고 LLM을 갈아 넣으며 남긴 중간 로그를 더 좋아합니다. 실패와 구린 결과도 숨기지 않습니다.

Note #01

가계부에 '미래의 나'가 없다

카드 내역을 LLM이 요약해 주면 좋을 줄 알았습니다. 대신 카페인을 줄이라고 잔소리만 늘었습니다. 지출을 '향후 영향'으로 재분류해보니 그제서야 행동이 움직였습니다.

다음 달 데이터 보는 중. 잔소리 강도는 30% 줄였습니다.

Note #02

회의록이 너무 착하다

LLM이 정리한 회의록은 분쟁을 모두 삭제했습니다. 덕분에 중요한 갈등이 사라졌죠. '합의 안 된 항목'만 따로 추출해 슬랙에 던졌더니 회의가 덜 길어졌습니다.

갈등 추출 정밀도 0.78. 더 비꼬는 톤 연구 중.

Note #03

루틴 추천이 체력보다 욕망이 앞섰다

웨어러블 데이터 없이 LLM이 만든 아침 루틴은 인간을 로봇으로 취급했습니다. 심박/수면 데이터를 붙이고 '가능한가?' 점수를 매기니 준수율이 올랐습니다.

준수율 +19%. 점수 계산은 아직 허술.

Note #04

알림을 줄였더니 데이터도 죽었다

이벤트 감지로 꼭 필요한 알림만 보내자고 필터를 걸었더니, 사용자 로그가 함께 줄었습니다. '왜 안 눌렀는지'를 묻는 마찰 없는 설문을 붙여 다시 학습시키는 중입니다.

응답률 14% → 31%. 행동 변화는 아직 미지수.

Experiments

문제, 가설, 모델, 실험

문제–가설–모델–실험–결과를 반복합니다. LLM도 데이터도 증거가 없으면 그냥 의견일 뿐입니다.

Problem
LLM이 고객 티켓을 엉뚱한 태그로 분류해 37%가 재작업으로 돌아옵니다.
Hypothesis
제품 스키마를 RAG로 물리고 '모르면 unknown'으로 버리면 헛발질이 줄어들 것.
Experiment
A: 기존 zero-shot 태깅. B: 제품 스키마+FAQ 벡터 검색 붙이고 abstention 설정. 두 그룹 모두 2주 측정(정확도, 재작업률, 평균 처리시간).
Result
정확도 +19pt, 재작업률 37%→11%, 처리시간 +6초. 속도는 조금 느려졌지만 팀이 다시 신뢰하기 시작했습니다.
Problem
개인화 루틴 추천 푸시는 72%가 무시합니다.
Hypothesis
웨어러블·캘린더 데이터로 시간/피로 비용을 예측해 함께 보여주면 덜 거부할 것.
Experiment
A: "지금 루틴 시작하세요" 단순 푸시. B: "15분, 예측 피로 -8%, 일정 충돌 없음"과 함께 전송. 4주 AB.
Result
시작율 2.4배, 중도 이탈 -22%. 데이터 연결 실패 케이스가 늘어 리트라이 로직 추가 예정입니다.
Problem
팀이 대시보드를 안 봅니다. 월간 보고서도 패스합니다.
Hypothesis
숫자만 요약한 LLM 리포트가 아니라, 지난주 대비 변동/원인/다음 실험을 한 페이지로 슬랙에 던지면 읽을지도.
Experiment
매주 금요일, 주요 메트릭 diff를 LLM이 문장화하고 근거 링크를 붙여 전송. 열람률, 회의 시간 변화를 6주 관찰.
Result
열람률 3배, 주간 회의 18분 단축. 해석 오류 2건 발생해 '불확실도' 섹션을 추가했습니다.
