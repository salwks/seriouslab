Index Guard — 흔들림 없는 투자 루틴

급등주 눌러담고 도파민에 취했다가 계좌가 식고, 단타로 번 돈을 단타로 두 배 잃고, 급락에 묶여 물타다 전재산이 희미해진 경험. 다들 한 번쯤 있습니다. Index Guard는 그 한순간을 심리 데이터로 끊어주는 iOS 앱입니다.

① 알림 울림: 거래량 폭증, 뉴스 도배, 손가락이 먼저 움직일 때
② 멈춤: F&G 지수와 이벤트 캘린더가 지금이 공포/탐욕 구간인지 먼저 보여줍니다
③ 실행: Daily Guard 체크리스트와 내가 정한 규칙으로 매수/매도 전에 숨 고르기

하루 1분 체크

"지금 사면 2배" 속삭임은 60초 타이머와 체크리스트로 끊습니다.

실시간 심리 + 이벤트

거래량 알림보다 먼저 F&G와 이벤트를 던져서, 손가락이 아니라 뇌가 먼저 움직이게.

월간 리포트

연속 성공일·완료율 그래프가 "내 멘탈은 내려갔지만 루틴은 올랐다"를 숫자로 증명.

나만의 규칙

"뉴스·거래량 폭발 시 24시간 대기" 같은 물타기 금지 규칙을 저장, Pro는 무제한/복원.

원페이지 배치 가이드

  • 헤더 우측 CTA 2버튼(Primary/Outline) 스티키 유지.
  • 히어로: 메시지+서브+앱 목업, CTA 반복 배치.
  • 기능 3~4카드 → 신뢰(데이터 출처/로컬 저장) → 구독 플랜(Monthly/Yearly).
  • FAQ/문의 버튼을 푸터 바로 위에 배치해 유입 손실 최소화.

톤 & UI

  • 딥 네이비/차콜 배경 + 라임 포인트, CTA는 그라데이션.
  • 1200px 컨테이너, 섹션 패딩 100px, 12그리드 4-4-4 카드.
  • 모바일: CTA 상단 고정, 카드 1열 스택, 여백 넉넉히.
  • 숫자/지표는 모노스페이스, 아이콘은 라인 스타일로 대비.

About Serious Lab

생활을 덜 귀찮게 만드는 데이터 연구팀

우리는 "원래 이렇게 쓰는 거예요"를 듣는 순간, 어디서 시간과 에너지가 새는지 기록부터 합니다. 불편을 구경만 하느니, 로그를 남겨 실험 재료로 삼는 편이 낫다고 믿습니다.

도구와 자동화를 즐겨 쓰지만, 판단과 우선순위는 사람이 내립니다. 숫자가 나아지지 않으면 새 도구도 의미 없습니다.

목표는 거창한 혁신보다 '조금 덜 귀찮은 하루'입니다. 실험과 기록을 쌓아, 생활이 실제로 나아졌다는 증거를 남깁니다.

생활 계측
시간·에너지·돈이 어디서 새는지 먼저 기록합니다.
도구는 수단
자동화와 모델은 빠른 손일 뿐, 결정권은 사람에게 있습니다.
사람이 브레이크
자동화가 미끄러지면 즉시 멈추고, 사람의 판단으로 다시 세웁니다.

Philosophy

Instrument, Experiment, Decide

문제를 감으로만 보지 않습니다. 계측 → 가설 → 실험 → 회복/개선률 확인의 루프를 반복합니다. 도구는 의심하고, 숫자 앞에서는 냉정합니다.

01

계측부터

현재 얼마나 안 좋은지 모르면, 좋아졌는지도 모릅니다. 로그와 베이스라인을 먼저 만듭니다.

02

도구는 인턴

자동화와 모델은 초안과 정리에 좋지만, 결정권은 주지 않습니다.

03

사람이 브레이크

Human-in-the-loop는 의무입니다. 모델이 엉뚱한 소리하면 즉시 끊고, 학습 데이터에 다시 반영합니다.

04

숫자로 선택

기분보다 개선폭·회복률·재현성을 봅니다. 움직이지 않는 아이디어는 미련 없이 버립니다.

Lab Notes

실패와 개선 로그를 숨기지 않습니다

결과물 자랑보다 과정과 실패를 드러낸 로그를 더 좋아합니다. 실험은 성공보다 회고가 재료가 되니까요.

Note #01

가계부가 내일을 반영하지 않는다

카드 내역을 잘 정리해도 "미래의 나"가 없다는 사실을 깨달았습니다. 지출을 '향후 영향'으로 재분류하니 그제서야 행동이 조금 움직였습니다.

다음 달 데이터 수집 중. 잔소리 강도는 30% 줄였습니다.

Note #02

회의록이 갈등을 지운다

자동 정리된 회의록은 분쟁을 모두 지워버렸습니다. '합의 안 된 항목'만 따로 추출해 슬랙에 던지니 회의가 덜 늘어졌습니다.

갈등 추출 정밀도 0.78. 어조는 더 다듬는 중.

Note #03

루틴 추천이 체력보다 욕망이 앞섰다

데이터 없이 만든 아침 루틴은 사람을 로봇 취급했습니다. 심박/수면 데이터를 붙이고 '가능한가?' 점수를 매기니 준수율이 올랐습니다.

준수율 +19%. 점수 계산은 아직 허술합니다.

Note #04

알림을 줄였더니 데이터도 줄었다

꼭 필요한 알림만 보내자고 필터를 걸었더니, 사용자 로그가 함께 줄었습니다. '왜 안 눌렀는지'를 묻는 짧은 설문을 붙여 다시 학습 중입니다.

응답률 14% → 31%. 행동 변화는 아직 관찰 중.

Experiments

문제, 가설, 실험, 증거

문제–가설–실험–결과를 반복합니다. 도구도 데이터도 증거가 없으면 의견일 뿐입니다.

Problem
자동 태깅이 고객 티켓을 엉뚱하게 분류해 37%가 재작업으로 돌아옵니다.
Hypothesis
제품 스키마와 FAQ를 검색해 근거를 붙이고, 모르면 모른다고 답하게 하면 헛발질이 줄 것.
Experiment
A: 기존 zero-shot 태깅. B: 제품 스키마+FAQ 벡터 검색 붙이고 abstention 설정. 두 그룹 모두 2주 측정(정확도, 재작업률, 평균 처리시간).
Result
정확도 +19pt, 재작업률 37%→11%, 처리시간 +6초. 속도는 조금 느려졌지만 팀이 다시 신뢰하기 시작했습니다.
Problem
개인화 루틴 추천 푸시는 72%가 무시합니다.
Hypothesis
웨어러블·캘린더 데이터로 시간/피로 비용을 예측해 함께 보여주면 덜 거부할 것.
Experiment
A: "지금 루틴 시작하세요" 단순 푸시. B: "15분, 예측 피로 -8%, 일정 충돌 없음"과 함께 전송. 4주 AB.
Result
시작율 2.4배, 중도 이탈 -22%. 데이터 연결 실패 케이스가 늘어 리트라이 로직 추가 예정.
Problem
팀이 대시보드를 안 봅니다. 월간 보고서도 패스합니다.
Hypothesis
숫자만 나열한 리포트 대신, 지난주 대비 변동/원인/다음 실험을 한 페이지로 슬랙에 던지면 읽을지도.
Experiment
매주 금요일, 주요 메트릭 diff를 자동 요약해 근거 링크와 함께 전송. 열람률, 회의 시간 변화를 6주 관찰.
Result
열람률 3배, 주간 회의 18분 단축. 해석 오류 2건 발생해 '불확실도' 섹션을 추가했습니다.

Labs

Model, Not Oracle
