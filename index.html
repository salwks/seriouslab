기술과 이론을 하나로, 덜 피곤한 하루를 만드는 방법을 실험합니다.

하루를 괜히 피곤하게 만드는 지점이 보이면, 그냥 참고 넘기지 않고 로그부터 남깁니다.
데이터를 붙이고 작은 실험을 반복하면서, 무엇을 줄이고 어디를 자동화할지 결정합니다.
도구는 수단일 뿐이고, 마지막 우선순위와 기준은 사람이 정합니다.
숫자로 “조금 덜 피곤해졌다”를 확인하는 쪽을 택합니다.

Index Guard — 손가락보다 멘탈을 먼저 잡는 앱

급등에 홀려 눌러담고, 단타로 번 돈을 단타로 날리고, 물타다 계좌가 안 보이게 된 경험. 대부분 이미 한 번씩은 겪었습니다. Index Guard는 그 “한 번 더”를 심리 지표와 이벤트 데이터로 끊어보자는 iOS 앱입니다.

① 알람: 거래량 폭증, 뉴스 도배, 손가락이 먼저 움직일 때 알림이 뜹니다.
② 멈춤: Fear & Greed 지수와 이벤트 캘린더로, 지금이 공포/탐욕 구간인지 먼저 보여줍니다.
③ 실행: Daily Guard 체크리스트와 내가 박제해둔 규칙으로, 매수/매도 전에 한 번 더 숨을 고릅니다.

하루 1분 체크

“지금 안 사면 바보다”라는 생각은, 60초 타이머와 체크리스트로 한 번 걸러냅니다.

심리 + 이벤트 한 번에

거래량 알림보다 먼저 F&G와 일정(옵션 만기, FOMC 등)을 보여 줘서, 손가락보다 뇌가 먼저 움직이게 합니다.

월간 리포트

연속 성공일·완료율 그래프로, “계좌는 요동쳐도 루틴은 버티고 있는지”를 숫자로 보여줍니다.

나만의 규칙

“뉴스/거래량 폭발 시 24시간 대기” 같은 규칙을 저장해 둡니다. Pro에서는 규칙을 여러 개 만들어 두고, 나중에 후회 없이 다시 돌아볼 수 있습니다.

원페이지 배치 가이드

  • 헤더 우측 CTA 2버튼(Primary/Outline) 스티키 유지.
  • 히어로: 메시지+서브+앱 목업, CTA 반복 배치.
  • 기능 3~4카드 → 신뢰(데이터 출처/로컬 저장) → 구독 플랜(Monthly/Yearly).
  • FAQ/문의 버튼을 푸터 바로 위에 배치해 유입 손실 최소화.

톤 & UI

  • 딥 네이비/차콜 배경 + 라임 포인트, CTA는 그라데이션.
  • 1200px 컨테이너, 섹션 패딩 100px, 12그리드 4-4-4 카드.
  • 모바일: CTA 상단 고정, 카드 1열 스택, 여백 넉넉히.
  • 숫자/지표는 모노스페이스, 아이콘은 라인 스타일로 대비.

About Serious Lab

생활과 일을 조금 덜 피곤하게 만드는 방법을 연구합니다

하루를 괜히 피곤하게 만드는 지점이 보이면, 그냥 적응하지 않고 기록부터 합니다. "원래 다 이렇게 쓰는 거예요"라는 말을 들으면, 어디서 시간과 에너지가 새는지 로그로 남깁니다. 그 불편은 다음 실험의 재료가 됩니다.

도구와 자동화를 좋아하지만, 우선순위와 책임은 사람 쪽에 둡니다. 그래프가 나아지지 않으면 새 도구도 그냥 장난감일 뿐입니다.

목표는 거창한 혁신보다 ‘조금 덜 피곤한 하루’입니다. 실험과 기록을 쌓아, 실제로 나아졌다는 증거를 남기는 쪽을 택합니다.

생활 계측
시간·에너지·돈이 어디서 새는지 먼저 기록합니다. 느낌이 아니라 로그로 봅니다.
도구는 수단
자동화와 모델은 일을 대신하는 손입니다. 결정권까지 넘기지는 않습니다.
사람이 브레이크
자동화가 미끄러지면 즉시 멈춥니다. 마지막 브레이크는 항상 사람이 밟습니다.

Philosophy

Measure, Mess With It, Decide

문제를 감으로만 보지 않습니다. 계측 → 가설 → 실험 → 회복·개선률 체크까지 한 루프를 돌립니다. 도구도 의심하고, 숫자 앞에서는 내 아이디어부터 먼저 의심합니다.

01

계측부터

지금 얼마나 안 좋은지 모르면, 좋아졌는지도 모릅니다. 로그와 베이스라인을 먼저 만든 뒤에야 손을 댑니다.

02

도구는 인턴

자동화와 모델은 초안 정리와 반복 작업에 좋습니다. 대신, 결론을 맡기지는 않습니다. 그건 아직 사람 몫입니다.

03

사람이 브레이크

Human-in-the-loop는 옵션이 아니라 안전장치입니다. 모델이 엉뚱한 답을 내면, 거기서 바로 잘라내고 왜 그랬는지 다시 학습에 반영합니다.

04

숫자로 선택

기분보다 개선폭·회복률·재현성을 봅니다. 움직이지 않는 아이디어는 애정 있어도 미련 없이 버립니다.

Lab Notes

결과보다, 삽질 로그를 더 중시합니다

완성된 결과물 자랑보다, 중간에 어디서 막혔는지 남긴 로그를 더 믿습니다. 실험은 성공보다 회고가 다음 버전의 재료가 되니까요.

Note #01

가계부가 내일을 반영하지 않는다

카드 내역을 정리해도 “미래의 나”는 여전히 안 보였습니다. 지출을 ‘향후 영향’ 기준으로 다시 묶으니, 그제야 행동이 조금 바뀌었습니다.

다음 달 데이터 수집 중. 잔소리 강도는 30% 낮췄습니다.

Note #02

회의록이 갈등을 지운다

자동 정리된 회의록은 갈등과 보류 안건들을 전부 매끄럽게 지워버립니다. '합의 안 된 항목'만 따로 추출해 슬랙에 던지니, 회의 시간이 덜 늘어졌습니다.

갈등 추출 정밀도 0.78. 어조는 아직 덜 인간적이라 손보는 중입니다.

Note #03

루틴 추천이 체력보다 욕망이 앞섰다

데이터 없이 만든 아침 루틴은 사람을 로봇 취급했습니다. 심박·수면 데이터를 붙이고 ‘가능한가’ 점수를 매기니 준수율이 조금은 사람답게 올랐습니다.

준수율 +19%. 점수 계산식은 아직 대충입니다.

Note #04

알림을 줄였더니 데이터도 줄었다

꼭 필요한 알림만 보내자고 줄였더니, 사용자 로그도 같이 사라졌습니다. '왜 안 눌렀는지'를 묻는 짧은 설문을 붙여, 다시 데이터부터 모으는 중입니다.

응답률 14% → 31%. 행동 변화는 아직 지켜보는 중입니다.

Experiments

문제, 가설, 실험, 그리고 증거

문제–가설–실험–결과. 이 네 칸이 안 채워지면 그건 아직 의견일 뿐입니다. 도구도, 데이터도, 증거가 쌓이기 전까지는 가설 취급합니다.

Problem
자동 태깅이 고객 티켓을 엉뚱하게 묶어서, 37%가 재작업으로 돌아옵니다.
Hypothesis
제품 스키마와 FAQ를 검색해 근거를 붙이고, 모를 땐 모른다고 말하게 하면 헛발질이 줄어들 것이다.
Experiment
A: 기존 zero-shot 태깅. B: 제품 스키마 + FAQ 벡터 검색을 붙이고, 애매하면 태깅을 포기하도록 설정. 두 그룹의 정확도, 재작업률, 처리 시간을 2주씩 비교했습니다.
Result
정확도 +19pt, 재작업률 37% → 11%, 처리 시간 +6초. 속도는 조금 느려졌지만, 팀이 다시 시스템을 믿기 시작했습니다.
Problem
개인화 루틴 추천 푸시는 72%가 무시합니다.
Hypothesis
웨어러블·캘린더 데이터로 시간/피로 비용을 같이 보여주면 덜 거부할 것이다.
Experiment
A: “지금 루틴 시작하세요” 단순 푸시. B: “15분, 예측 피로 -8%, 일정 충돌 없음”을 함께 전송. 4주간 AB 테스트를 진행했습니다.
Result
시작율 2.4배, 중도 이탈 -22%. 대신 데이터 연결 실패 케이스가 늘어, 리트라이 로직을 다시 설계 중입니다.
Problem
대시보드를 아무도 안 봅니다. 월간 보고서도 옵션 취급입니다.
Hypothesis
숫자 나열 대신, 지난주 대비 변동·원인·다음 실험을 한 페이지로 슬랙에 던지면 읽을지도 모른다.
Experiment
매주 금요일, 주요 메트릭 diff를 자동 요약해 근거 링크와 함께 전송했습니다. 열람률과 회의 시간을 6주간 관찰했습니다.
Result
열람률 3배, 주간 회의 18분 단축. 해석 오류 2건이 생겨 ‘불확실도’ 섹션을 따로 추가했습니다.

Labs

관심사별로 쪼개서 실험합니다

하나의 거대한 조직이라기보다, 관심사별로 실험하는 작은 랩들의 묶음에 가깝습니다. 계측하고, 도구로 실험하고, 실패도 기록으로 남깁니다. 그게 나중에 제일 쓸모가 있어서요. 최종 목표는 멋진 화면이 아니라, 하루가 조금 덜 피곤해지는 것입니다. 화려한 UI보다, 피로도·시간·손이 덜 가는 쪽으로 지표가 움직이는지를 봅니다.

계측 광

Instrument Everything

사소한 습관도 이벤트로 남깁니다. 측정 불가면 개선 불가라고 생각합니다.

모델은 도구

Model, Not Oracle

자동화와 모델은 정리와 분류용입니다. 답을 대신 믿어주는 존재는 아닙니다.

실험 집착

Hypothesis to Log

모든 변화는 가설과 로그로 시작하고 끝납니다. 실패도 아카이브에 그대로 남겨둡니다.

생활 임팩트

Life Impact

제품보다 생활이 조금 덜 피곤해지는 게 목표입니다. 화려한 UI보다 지표 움직임을 봅니다.
